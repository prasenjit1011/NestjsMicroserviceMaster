  • What is Laravel?
    Laravel is a free, open-source PHP web framework used to develop web applications following the MVC (Model-View-Controller) pattern. It provides elegant syntax, routing, authentication, caching, and more.
  • What are the features of Laravel?
    Key features include:
    - MVC architecture
    - Eloquent ORM
    - Routing
    - Middleware
    - Blade templating engine
    - Artisan CLI
    - Security (CSRF, XSS protection)
    - Task scheduling and queues
  • What is MVC architecture?
    MVC stands for Model-View-Controller.
    - Model: Manages data and business logic
    - View: Handles presentation
    - Controller: Handles user input and updates Model/View
  • What are routes in Laravel?
    Routes define the URLs your application should respond to and map them to controllers or closures. They are defined in routes/web.php or routes/api.php.
  • What is the use of Composer in Laravel?
    Composer is a dependency manager for PHP. It is used in Laravel to install and manage packages and libraries required by the application.
  • What is a Service Provider in Laravel?
    Service Providers are the central place to configure and bootstrap application services. All service providers are registered in config/app.php in the providers array.
  • What is Middleware in Laravel?
    Middleware filters HTTP requests entering your application. It is used for tasks like authentication, logging, CORS, etc. You can assign middleware to routes or route groups.
  • What is CSRF protection in Laravel?
    CSRF (Cross-Site Request Forgery) protection prevents unauthorized commands from being performed. Laravel includes a CSRF token in every form, which is verified on submission.
  • What is Blade in Laravel?
    Blade is Laravel's templating engine. It allows you to use PHP logic in HTML using directives like @if, @foreach, @extends, @section.
  • How to install Laravel?
    You can install Laravel using Composer:
    composer create-project laravel/laravel example-app
  • What is the use of the .env file?
    The .env file stores environment-specific configuration like database credentials, app URL, and keys. It's used by Laravelâ€™s configuration system.
  • What are facades in Laravel?
    Facades provide a static interface to classes in the service container. For example, Cache::get() is a facade for the caching system.
  • How do you define a controller in Laravel?
    Use Artisan command:
    php artisan make:controller UserController
    This creates a controller in app/Http/Controllers.
  • What is the difference between web.php and api.php?
    - web.php: For web routes using sessions and CSRF protection.
    - api.php: For stateless API routes using the api middleware group.
  • What is a Request in Laravel?
    A Request is an object representing an HTTP request. You can access input data using:
    $request->input('key')
    or dependency-inject Illuminate\Http\Request.
  • How do you handle errors in Laravel?
    Laravel handles errors using the app/Exceptions/Handler.php class. You can render custom error responses or log errors using built-in logging facilities.
  • What is the use of the config directory?
    The config directory contains all of your application's configuration files such as app.php, database.php, mail.php, etc.
  • How to use session in Laravel?
    Laravel provides multiple session drivers. You can access sessions via:
    session(['key' => 'value']);
    session('key');
  • What is Laravel Mix?
    Laravel Mix is a wrapper around Webpack that compiles and optimizes assets like CSS, JS, and SASS. It's configured in webpack.mix.js.
  • What is a Helper function in Laravel?
    Helpers are global functions that provide shortcuts for common tasks, e.g., route(), asset(), url(), dd().
  • What is Eloquent ORM in Laravel?
    Eloquent ORM (Object Relational Mapper) is Laravel's built-in ORM that provides a simple ActiveRecord implementation for working with the database using models.
    Example:
    $users = User::where('status', 'active')->get();
  • What are Eloquent relationships?
    Eloquent relationships define associations between Eloquent models, such as:
    - One to One: hasOne
    - One to Many: hasMany
    - Many to Many: belongsToMany
    - Polymorphic: morphTo
    Example:
    public function posts() {
          return $this->hasMany(Post::class);
      }
  • How do you use Laravel migrations?
    Migrations are version control for your database schema. Create and run migrations:
    php artisan make:migration create_users_table
      php artisan migrate
    Define schema using Laravelâ€™s fluent schema builder.
  • What are seeders and factories?
    - Seeders populate test data into the database.
    - Factories define model blueprints for generating fake data.
    Example:
    User::factory()->count(5)->create();
  • What is Laravel Artisan?
    Artisan is the command-line interface included with Laravel. It provides commands for generating files and running tasks.
    Example:
    php artisan list
      php artisan make:controller UserController
  • How do you validate form data in Laravel?
    You can validate data in controllers using:
    $request->validate([
          'name' => 'required|max:255',
          'email' => 'required|email',
      ]);
    Or use Form Request classes with custom rules.
  • How to create and use custom Artisan commands?
    Create a custom command using:
    php artisan make:command CustomCommand
    Register it in app/Console/Kernel.php, then run it:
    php artisan custom:command
  • How to use Eloquent accessors and mutators?
    - **Accessors**: Format values when accessing model attributes.
    - **Mutators**: Modify values before saving to the database.
    Example:
    public function getNameAttribute($value) {
          return ucfirst($value);
      }
  • How do you use soft deletes in Laravel?
    Use SoftDeletes trait in your model:
    use Illuminate\Database\Eloquent\SoftDeletes;
      protected $dates = ['deleted_at'];
    Then run:

  • How to prevent mass assignment vulnerabilities?
    Use $fillable or $guarded properties in Eloquent models:
    protected $fillable = ['name', 'email'];
    This ensures only allowed fields can be mass-assigned.
  • What is the difference between authentication and authorization?
    - Authentication verifies user identity (e.g., login).
    - Authorization determines what actions a user can perform (e.g., admin access).
  • How does Laravel handle authentication?
    Laravel provides built-in authentication scaffolding using laravel/ui or laravel/breeze.
    Laravel uses Auth facade and auth middleware for login, registration, and session management.
  • What are guards in Laravel?
    Guards define how users are authenticated for each request. Configured in config/auth.php.
    Common guards: web (session), api (token).
  • What are providers in Laravel authentication?
    Providers define how users are retrieved from storage (usually the database). Defined in config/auth.php.
    Example: Eloquent or Database provider.

    Gates provide a simple closure-based approach to authorization for actions that arenâ€™t related to any model.
    Example:
    Gate::define('view-dashboard', function ($user) {
          return $user->isAdmin();

  • How do you restrict access to certain routes?
    You can use route middleware:
    Route::middleware(['auth', 'can:isAdmin'])->group(function () {
          Route::get('/admin', [AdminController::class, 'index']);
      });

  • What is API Resource in Laravel?
    API Resources transform models and collections into JSON responses. Example:
    php artisan make:resource UserResource
    Usage:

  • How to return JSON response in Laravel?
    Laravel provides helper methods for JSON responses:
    return response()->json(['message' => 'Success'], 200);

    Rate limiting helps control API abuse. Laravel uses the `ThrottleRequests` middleware.
    In routes/api.php:
    Route::middleware('throttle:60,1')->group(function () {
          Route::get('/user', function () {
              return auth()->user();
          });
      });

    - API Resources: single model transformation.

    return UserResource::collection(User::all());

    - api.php: routes for stateless APIs, uses api middleware group.

  • What is the use of service providers in Laravel?
